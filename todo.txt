可以进一步做的方向

  1. 前端多源切换 UI
  当前后端支持多平台，但前端（webroot/js/main.js）似乎仍默认走网易云。可以加一个音源切换按钮，让用户在前端选择
  kuwo/kugou/bilibili 等平台搜索。

  2. UNM 搜索功能增强
  unm.js 的 proxy('search') 直接返回 null（第208行），只在 search() 方法里做了搜索。可以让 UNM 也参与 proxy 层面的搜索
  fallback。

  3. 歌词/封面的多源 fallback
  UNM provider 的 lyric() 和 pic() 都返回 null（第157-165行），依赖下一个 provider 兜底。如果 gdstudio
  歌词也挂了，就没有第三层了。可以考虑用 Meting 补充歌词。

  4. 真正的 UNM 外部服务集成（可选）
  当前是内嵌 UNM 模块直接 require，优点是无外部依赖。但如果 @unblockneteasemusic/server 的模块结构变化，可能会
  break。todo.txt 提到的 npx unblockneteasemusic 启动独立服务的方案可以作为 fallback。

  5. 播放历史 / 最近播放
  数据库有 favorites 和 playlists，但没有 play_history 表。可以加一个播放记录功能。

  6. API 状态监控完善
  api_status 表和 /php/api_doubtful.php
  路由已就绪，但没有定时健康检查的逻辑。可以加一个定时任务自动检测各音源可用性并写入 api_status。
